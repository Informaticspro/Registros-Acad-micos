# Estructura Del Proyecto En Español

La carpeta `src` ahora esta organizada asi:

| Carpeta | Uso |
| --- | --- |
| `componentes` | Componentes reutilizables de interfaz y estructura. |
| `componentes/estructura` | Layout general, encabezado y barra lateral. |
| `componentes/interfaz` | Componentes UI pequeños como tarjetas y encabezados. |
| `componentes/registro` | Componentes visuales relacionados con QR/registro. |
| `datos` | Datos de prueba para modo desarrollo. |
| `estilos` | CSS global de la aplicacion. |
| `infraestructura` | Configuracion de entorno y cliente Supabase. |
| `modulos` | Funcionalidades principales de la aplicacion. |
| `modulos/administracion` | Usuarios del sistema. |
| `modulos/asistencia` | Escaneo y marcaje de asistencia. |
| `modulos/autenticacion` | Login y sesion. |
| `modulos/eventos` | CRUD y detalle de eventos. |
| `modulos/participantes` | Listado de participantes registrados. |
| `modulos/registro` | Registro publico, consulta de QR y formularios dinamicos. |
| `paginas` | Paginas globales como no encontrado. |
| `rutas` | Router, ruta protegida y control por rol. |
| `servicios` | Acceso a Supabase y funciones serverless. |
| `tipos` | Tipos TypeScript del dominio y Supabase. |
| `utilidades` | Funciones auxiliares para errores, fechas y QR. |

No subi estos cambios a GitHub. Quedaron locales hasta que indiques lo contrario.
